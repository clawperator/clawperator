# Docs Refactor Plan

## Status

Blocked on: `tasks/api/refactor/` (all phases complete). Writing docs against a
moving API surface is waste. These PRDs execute only after the CLI command
surface is stable.

---

## Background

The docs site (`docs.clawperator.com`) has two problems:

1. **Entry points and discoverability.** Agents cannot find docs from install
   output or error messages. First-run guides compete rather than forming a
   hierarchy. The `AGENTS.md` orientation file does not exist yet.

2. **Information architecture.** The site mixes user journeys, personas, content
   types, and internal architecture into a single incoherent hierarchy with
   competing entry points and duplicate pages. The content is accurate; the
   structure is not.

These are separate problems with a sequencing dependency: fix entry points first
(PRD-1), then restructure the site around those entry points (PRD-2).

## Implementation Notes for Agents

1. All command references, CLI examples, and flag names must use the new flat
   command surface from the API refactor. Do not reference `action click`,
   `observe snapshot`, `--device-id`, or `--output json`.

2. PRD-1's `~/.clawperator/AGENTS.md` template must use new commands:
   `clawperator snapshot`, `clawperator click --text "..."`, `--device`, etc.

3. PRD-2's reference section must document the new CLI surface, not the old one.

4. The `docsUrl` values hardcoded in `readinessChecks.ts` (added by PRD-1) may
   need updating if PRD-2 moves pages. Search `apps/node/src/domain/doctor/checks/`
   for `docsUrl` values.

## Landing Page Update (blocked on API refactor)

After the API refactor ships, update the agent loop pseudocode on the landing
page (`sites/landing/app/page.js`, the `.loop-example` section). The current
pseudocode uses invented syntax (`open_app()`, `snapshot_ui()`,
`scroll_and_click()`) that does not reflect the actual API and undersells the
product.

Target replacement:

```javascript
const clawperator = require("clawperator");
await clawperator.open("com.google.home");
let ui = await clawperator.snapshot();
await clawperator.click({ text: "Climate" });
ui = await clawperator.snapshot();
await clawperator.scrollUntil({ text: "Living room", click: true });
ui = await clawperator.snapshot();
if (ui.nodes.find(n => n.text === "Off")) {
  await clawperator.click({ text: "Turn on" });
}
```

This is real Node API code, not pseudocode. Every line is self-explanatory
without documentation. The snippet demonstrates: app launch, UI observation,
text-based element targeting, scroll-until-found, conditional interaction -
the full agent loop in 10 lines.

Note: `scrollUntil` ships in the API refactor Phase 5A.
